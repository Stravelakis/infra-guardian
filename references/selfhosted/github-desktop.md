# GitHub — Codespaces & Desktop

Two ways we interact with GitHub repos, for two different moments.

## GitHub Codespaces (cloud)
- A full dev environment **in the browser** (this workspace is one).
- No local setup — the machine, terminal, and editor live on GitHub.
- Use for: quick edits, running scripts, working from any machine.

## GitHub Desktop (local app)
- Desktop GUI for Git — commit, branch, push, and pull **without the
  command line**.
- Download: https://desktop.github.com/download/
- Use for: working on files on your own computer, visual diff review,
  and drag-and-drop commits.

## When to use which
| Situation | Tool |
|-----------|------|
| On a borrowed/any machine | Codespaces |
| Heavy local files / offline work | Desktop |
| Want a visual, no-terminal Git workflow | Desktop |
| Need the full Linux shell + scripts | Codespaces |

Both push to the **same repo** — pick whichever fits where you're sitting.
