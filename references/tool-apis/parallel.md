# Parallel

**In one line:** A web-research API built for AI — you ask a question,
it searches the live web and returns clean, source-backed answers your
app can use directly.

## What it's for
- Give AI apps up-to-date facts from the web (instead of stale training data).
- Get answers with citations, so you can trust and verify them.
- Do deep, multi-source research tasks via an API call.

## How it fits your stack
- Same category as Exa and Tavily (see their files) — a "search for AI" tool.
- Called from your apps/agents (n8n, Goose, LibreChat) when they need
  fresh information.
- Connect via an **API key** in `.env`.

## Key facts
- Hosted API; usage-based pricing.
- Auth: API key.
- Returns structured, citation-backed results (not just raw links).
- Offers different "depth" levels — quick lookups vs. thorough research.

## Watch out for
- External paid service — watch usage costs on heavy research loops.
- Compare against Exa/Tavily for your specific use; strengths differ.

## New glossary terms
| Term | In plain English |
|------|------------------|
| API | A doorway that lets one program request things from another. |
| citation | A pointer to the source a fact came from. |
| structured output | Data returned in a tidy, predictable shape a program can read. |
